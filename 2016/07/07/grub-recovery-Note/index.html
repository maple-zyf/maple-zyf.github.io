修复grub

重装系统或从备份中恢复系统导致grub出错，开机进入不了系统和正常的grub界面.

开机时显示: 

error:unknow filesystem
grub rescue>

解决方法

用ls命令查看分区，由grub rescue>恢复到grub> 

grub rescue>ls
(hd0) (hd0,msdos9) (hd0,msdos8) (hd0,msdos7) (hd0,msdos6) (hd0,msdos5) (hd0,msdos1)
//哪个是你的系统盘呢？　这个要试试，可以用Tab补全来看看文件，从而找到正确的盘号.
grub rescue>set root=(hd0,msdos7)
grub rescue>set prefix=(hd0,msdos7)/boot/grub
grub rescue>insmod /boot/grub/i386-pc/normal.mod 
//启动normal启动

以上命令正确则下一行命令会变为高亮 

grub rescue>normal

此时进入grub菜单，但无法进入任何系统。 这时，要用命令进入一个存在的linux系统: 按c键进入命令指示符模式 

grub >set root=hd0,msdos7
grub >set prefix=(hd0,msdos7)/boot/grub
grub >linux /vmlinuz-xxx-xxx root=/dev/sda7 
//里边的xxxx可以按Tab键，如果有acpi问题,在最后加一句acpi=off
grub >initrd /initrd.img-xxx-xxx
grub >boot

此以成功启动linux。 在终端里运行 

sudo update-grub    
//用这个命令,不要再用 grub-mkconfig了(总出些/sda3  /sda6分不清楚的问题).
重建grub到第一硬盘mbr
sudo grub-install /dev/sda

用grub直接启动iso镜像

step1: debian上安装grub-imageboot( 会自动生成一个/boot/memdisk ) 或者是下载syslinux包 找到里边的memdisk复制到/boot

step2: 在/etc/grub.d/40custom 添加： 下边是我自己的配置，因为memdisk在sda3, 而iso文件在sda1. 

menuentry "Load Rescue.iso" {
  echo 'Loading Memdisk...'
  linux16 (hd0,3)/boot/memdisk iso raw
	echo 'Loading ISO...'
  initrd16 (hd0,6)/zengyingfeng/c_language/linux_C/ISO/Rescue.iso
}
menuentry "Load ubuntu-live.iso" {
  set root=(hd0,6)
  loopback loop /zengyingfeng/c_language/linux_C/ISO/ubuntu-12.04.4-desktop-i386.iso
  linux (loop)/casper/vmlinuz boot=casper iso-scan/filename=/zengyingfeng/c_language/linux_C/ISO/ubuntu-12.04.4-desktop-i386.iso
  initrd (loop)/casper/initrd.lz
}

step3: 用grub将新的配置写入MBR 

sudo update-grub
Found memdisk: /boot/memdisk
Imagepath /boot/images not found
done
//这样的回显是正常的
